サイドバー、目次、ページタイトルは、次の 2 つから組み立てられます。

- 各 `.md` ページの先頭にある **フロントマター**
- 任意のフォルダに置いた **`meta.json`**

どちらも fumadocs と同じスキーマなので、既存の fumadocs コンテンツツリーをそのまま持ち込めます。

## フロントマター

ページごとに、自身のメタデータを書けます。

```md
---
title: Getting Started
description: Preview any directory of Markdown in seconds.
icon: Rocket
---

Page body starts here.
```

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `title` | `string` | 省略すると、拡張子を除いたファイル名が使われます。 |
| `description` | `string` | インデックスページと `<meta>` の description に使われます。 |
| `icon` | `string` | [Lucide](https://lucide.dev) のアイコン名（例: `FileText`）。 |

ページには、ほかの任意のフィールド（`status`、`author` など）も書けます。これらはナビゲーションには影響しませんが、ページ冒頭のメタデータ表として見せることができます。詳しくは設定の [`frontmatter`](./configuration.ja.md#frontmatter) を参照してください。

## `meta.json`

フォルダに `meta.json` を置くと、そのフォルダがサイドバーにどう並ぶかを指定できます。

```json
{
  "title": "Guides",
  "description": "Long-form how-to articles",
  "icon": "BookOpen",
  "pages": [
    "getting-started",
    "configuration",
    "---Advanced---",
    "navigation",
    "markdown"
  ],
  "defaultOpen": true
}
```

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `title` | `string` | サイドバーでのフォルダ名。 |
| `description` | `string` | ツールチップや説明。 |
| `icon` | `string` | Lucide のアイコン名。 |
| `pages` | `string[]` | 子要素の並び順。各要素は拡張子を除いたファイル名か、フォルダ名です。 |
| `defaultOpen` | `boolean` | フォルダを最初から開いておきます。 |
| `root` | `boolean` | このフォルダを、独自のヘッダーを持つ独立したセクションとして扱います。 |

### `pages` で並び順を決める

`pages` がなければ、要素はアルファベット順に並びます。順番を決めたいときは、明示的に列挙します。

```json
{
  "pages": ["index", "install", "first-steps"]
}
```

ここに書くのはタイトルではなく、ファイル名やサブフォルダ名です。`"..."` を使うと、列挙しなかった要素をまとめて差し込めます。

```json
{
  "pages": ["index", "install", "..."]
}
```

### 区切り

`---` で囲んだ文字列は、要素のあいだに置くクリックできないラベルになります。

```json
{
  "pages": [
    "intro",
    "---Setup---",
    "install",
    "config"
  ]
}
```

`"---"` だけを書くと、細い区切り線になります。

## 検証エラー

`meta.json` の解析に失敗したとき（JSON が壊れている、`"title": 123` のように型が違うなど）や、フロントマターの検証に通らないときは、黙って見過ごされることはありません。すべてのページの先頭にエラーバナーが出て、問題のファイルと内容を知らせます。フロントマターが不正なページも、既定値にフォールバックしてレンダリングは続きます。ファイルを直せば、次のホットリロードでバナーは消えます。

## マルチプロジェクトのタブ

[設定](./configuration.ja.md#projects) でプロジェクトを 2 つ以上指定すると、それぞれがサイドバー上部の **タブ** になります。タブを切り替えると、そのプロジェクトのツリーだけが表示されます。プロジェクトのスラッグ（`name` から作られます）は、すべてのページ URL の先頭に付きます（例: `/guides/install`）。

```json
{
  "projects": [
    { "name": "Guides", "dir": "./docs" },
    { "name": "Blog",   "dir": "./blog" }
  ]
}
```

各プロジェクトのトップレベルにある `meta.json` は、自動でセクションルート（`"root": true`）として扱われます。自分で書く必要はありません。

### タブのトップページ

タブのトップ（`/{slug}`）に何が出るかは、プロジェクト直下に `index.md` があるかどうかで決まります。

- **`index.md` がある場合** — それがタブのトップページになります。
- **`index.md` がない場合** — すべてのページがサブフォルダにあっても、プロジェクトのページ一覧が自動で生成され、タブのトップに表示されます。これは [インデックスページ](#インデックスページ) でいう「クリックできないグループ」にはなりません。

## アイコン

フロントマターでも `meta.json` でも、[Lucide](https://lucide.dev) のアイコン名を文字列で指定できます。

```yaml
icon: FileText
```

指定すると、**サイドバー** の該当する項目のラベルの左に、小さなアイコンが付きます。フロントマターの `icon` はそのページのリンクに、`meta.json` の `icon` はそのフォルダ（またはセクションルート）のラベルに付きます。アイコンが出るのはサイドバーだけで、ページ本文の見出しには影響しません。指定しなければ何も出ません。

アイコン名は [Lucide](https://lucide.dev) のものと一致している必要があります（大文字・小文字も区別します）。見つからない名前を指定すると、その項目にはアイコンが付かず、起動時に警告が出ます。アイコンは必要になったときに読み込まれ、実際に使うものだけが取り込まれます。

## インデックスページ

フォルダ名そのものの URL（例: `/guides`）を開いたとき、`guides/index.md` があればそれが開きます。これがフォルダの **トップページ** です。

`index.md` のないフォルダは、URL を持たない **クリックできないグループ** になります。フォルダ名はサイドバーに出ますが、クリックしても開閉するだけで、開くページはありません。

これと挙動が違うのは、プロジェクトタブの直下（[マルチプロジェクトのタブ](#マルチプロジェクトのタブ)）だけです。ここでは `index.md` がなくてもクリックできないグループにはならず、自動生成されたページ一覧がタブのトップに表示されます。
