sladocs の実行に必要なのは、Markdown の入ったディレクトリだけです。

## 必要なもの

- **Node.js 20 以降**（22 LTS を推奨）
- `.md` または `.mdx` ファイルの入ったディレクトリ

## 実行

```bash
npx sladocs
```

このコマンドは、カレントディレクトリを起点にドキュメントを集めてサイトを組み立て、プレビューします。別の場所を起点にしたいときはパスを渡します。

```bash
npx sladocs ./docs
```

プレビューサーバーが起動すると URL が表示されるので、ブラウザで開いてください。

```text
ready: http://localhost:8080/
```

`Ctrl-C` を押せば、ファイル監視と HTTP サーバーを終了できます。

> [!NOTE]
> 初回は sladocs を npm キャッシュにダウンロードするため、起動に数秒かかります。
> 2 回目以降はそのキャッシュを使い回すので、高速に起動します。

## 対象ページ

sladocs は、対象ディレクトリ配下の Markdown（`.md`、`.mdx`）ページを集めてサイトを組み立てます。ナビゲーション用の `meta.json` も併せて拾います。サイドバー・検索・ホットリロードは、いずれもこの集めたページが対象です。`.mdx` ファイルは Markdown としてのみ解釈し、JSX や `import` / `export` は評価せず、ただのテキストとして残します。

git リポジトリなら、`.gitignore` に書かれたファイルは自動で除外します。`node_modules` や `dist` などを除外するための追加の指定は不要です。git 管理下にないディレクトリでは、`node_modules` だけを除外します。

対象ファイルは [`include` / `exclude`](./configuration.ja.md#projects) で調整することもできます。

## ライブリロード

サーバーを起動したまま、サイトに表示されているファイルを編集して保存すると、ブラウザがその場で更新されます。

## 複数のプロジェクト

sladocs は、標準では 1 つのディレクトリを起点にサイトを組み立てます。パスを 2 つ以上渡すと、マルチプロジェクト表示に切り替わります。

```bash
npx sladocs ./docs ./packages/web
```

ディレクトリごとに、トップナビゲーションのタブ（ドロップダウン）ができます。タブの名前にはフォルダ名が使われますが、`sladocs.json` で変更できます（[設定](./configuration.ja.md) を参照）。

## 複数の言語

`sladocs.json` で使用する言語を宣言すると、サイドバーに言語の切り替えが現れます。ファイル名にロケールを付けること（`guide.md`、`guide.ja.md`）で、言語ごとに別のページを表示できます。

```json
{
  "i18n": { "languages": ["en", "ja"], "defaultLanguage": "en" }
}
```

ファイルの命名、URL の構成、切り替えのラベルについては [`i18n`](./configuration.ja.md#i18n) を参照してください。

## CLI リファレンス

```text
sladocs [dirs...] [options]
```

### 引数

| 引数 | 説明 |
| --- | --- |
| `dirs...` | サイトの起点にするディレクトリ。1 つ以上を指定でき、省略するとカレントディレクトリが対象になります。複数指定すると、それぞれがトップナビゲーションのタブになります。 |

`sladocs.json` に `projects` が書かれている場合は、CLI 引数を無視して設定を優先します（[設定](./configuration.ja.md) を参照）。

### オプション

| フラグ | 短縮 | 既定値 | 説明 |
| --- | --- | --- | --- |
| `--port <port>` | `-p` | `8080` | 待ち受けを始めるポート。使用中なら、空いているポートを順に探して繰り上げます。 |
| `--host <host>` | `-H` | `localhost` | バインドするアドレス。LAN から見えるようにするには `0.0.0.0` を指定します。 |
| `--no-watch` | – | – | ファイル監視とホットリロードを無効にします（既定では有効）。 |
| `--help` | `-h` | – | 使い方を表示します。 |
| `--version` | `-v` | – | インストール済みのバージョンを表示します。 |
