**sladocs** は、Markdown のディレクトリをコマンド 1 つでプレビューするサーバーです。サイドバー、目次、全文検索、ダークモードを備えた [Fumadocs](https://fumadocs.dev) の洗練された UI を、インストールも設定もなしですぐに使えます。

```files
project
├── docs
│   ├── index.md
│   ├── spec.md
│   └── design.md
├── README.md
```

```bash
npx sladocs
```

## いつ sladocs を使うか

コードと同じリポジトリに Markdown でドキュメントを置いているなら、それらを読むのに sladocs が適しています。ディレクトリ構造がそのままナビゲーションになり、全ページを横断して検索できるので、設計メモ・運用手順・仕様書といった複数ページにまたがるドキュメントを体系的に閲覧できます。

`meta.json` を追加で置くことで、ファイル名順ではなく読み手にとって自然な順序でページを並べたり、グループにまとめたりできます。

## できること

- GFM のレンダリング。コールアウト、コードブロック、インライン HTML など
- Mermaid のダイアグラム、KaTeX の数式、Shiki によるシンタックスハイライト
- 画像・HTML プロトタイプ・PDF などの静的アセット配信
- `meta.json` によるナビゲーション、ページ間リンクの解決、全文検索
- 保存時のホットリロード
- 複数のディレクトリをまとめて扱うマルチプロジェクト表示
- ドキュメントサイトの静的ビルド

## fumadocs-preview との違い

Markdown ディレクトリを手早く表示するツールとしては、Fumadocs にも公式 CLI の [fumadocs-preview](https://www.fumadocs.dev/docs/cli/preview) があります。役割が重なる部分もあるので、次の違いを押さえておくとよいでしょう。

- **設定してもインストール不要。** fumadocs-preview はカスタマイズに dev 依存としてのローカルインストールを前提とします。sladocs は設定を加えたあとも `npx` だけで完結します。
- **リンクはただの Markdown。** ページ間の相対リンクは書いたとおりに解決されます。書き換えも、ビルド時の特別な記法も要りません。
- **静的アセットもそのまま。** 画像はもちろん、HTML プロトタイプや PDF などへの相対リンクも直接配信します。
